Best Practices

How you make your Open API file is up to you, so long as there is an entry point, in a special folder dedicated to documentation/specification. This entire folder is going to be copied into the public/ directory, so do not use spec_dir: / unless you want your source code to be public!

Other than that, here is some general advice.

  1. Use Open API v2.0, not v3.0 (yet)
  2. Add a x-logo object because it looks nice
  3. Split the definitions into their own `docs/definitions/foos.yml` files and use $ref.
